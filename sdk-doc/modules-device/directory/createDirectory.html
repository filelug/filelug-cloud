createDirectory（尚未實作）
回上一層
模組 檔案目錄
說明 新增指定目錄
回傳值 被新增的目錄資訊
URL POST http://<domain>:<port>/crepo/directory
參數範例（格式：application/json）
{
  "symlink" : null,
  "name" : "電子雜誌",
  "parent" : "/Users/albert2013/Documents",
  "readable" : true,
  "writable" : true,
  "executable" : true,
  "type" : "DIRECTORY"
}
參數值說明
名稱 類型 說明
必要參數
parent string 父目錄絕對路徑。此路徑必須為encode後的路徑。
例如路徑「C:\\Documents and Settings\\Administrator\\My Documents」 經過encode後為「C%3A%5CDocuments%20and%20Settings%5CAdministrator%5CMy%20Documents」； 而路徑「/Users/clopuccino/課程資料」經過encode後為「%2FUsers%2Fclopuccino%2F%E8%AA%B2%E7%A8%8B%E8%B3%87%E6%96%99」。
URL encoding的規則詳見：HTML URL Encoding Reference，其中的「URL Encoding Functions」可用來encode URL。
name string 目錄名稱。此名稱必須為encode後的名稱。
例如「課程資料」經過encode後為「%E8%AA%B2%E7%A8%8B%E8%B3%87%E6%96%99」。
URL encoding的規則詳見：HTML URL Encoding Reference，其中的「URL Encoding Functions」可用來encode URL。
非必要參數
readable boolean 是否有「讀」此目錄的權限。若有權限設為true否則設為false。若不設定，則此權限繼承自其父目錄。
writable boolean 是否有「寫」此目錄的權限。若有權限設為true否則設為false。若不設定，則此權限繼承自其父目錄。
executable boolean 是否有「執行」此目錄的權限。若有權限設為true否則設為false。若不設定，則此權限繼承自其父目錄。
type string 目錄或檔案。此服務下的值固定為「DIRECTORY」
回傳值範例（格式：application/json）
{
    "symlink":false,
    "name":"每日五蔬果 癌症遠離我",
    "parent":"/Users/clopuccino/Documents",
    "readable":true,
    "writable":true,
    "executable":true,
    "hidden":false,
    "lastModified":"2013/03/08 14:51:55 GMT+0800",
    "type":"DIRECTORY",
    "contentType":"application/directory"
}
回傳值說明
屬性名稱 類型 說明
必要屬性
symlink boolean 是否為symbolic link。對此服務而言，由於是建立實體目錄，故此值永遠是false
name string 目錄名稱。
parent string 父目錄絕對路徑
readable boolean 是否有讀的權限
writable boolean 是否有寫的權限
executable boolean 是否有執行的權限
hidden boolean 是否為隱藏檔。 若指定的目錄名稱（URL encode前）是以「.」開頭，此值即為true，否則為false
lastModified string 最後修改日期。格式為「yyyy/MM/dd HH:mm:ss 'GMT'Z」
type string 目錄或檔案。目錄值為「DIRECTORY」、檔案值為「FILE」。 對此此服務而言，由於是目錄，因此值永遠是「DIRECTORY」
contentType string 對此此服務而言，由於type為「DIRECTORY」時，故此值永遠是「application/directory」。
非必要屬性
（無）
回傳錯誤訊息（HTTP Status）
訊息碼 說明
400 指定的目錄絕對路徑已存在於作業系統中，可能是目錄，也可能是現存的檔案
401 session 已過期（預設有效期間為 60 分鐘，自上一次使用此 session id 調用服務的時間開始計算）。
收到此回傳值，前端應調用服務 loginWithSessionId 取回新的 session id 後再重新調用此服務。
403 session 不存在。或者該 session 所對應的用戶不存在。
收到此回傳值，前端應提示用戶使用 Facebook Account Login 重新登入。
503 server 與 desktop 間尚未建立連線。
補充說明
  • 必須先登入系統，取得有效的session id，以作為此服務的 header「fsi」(對 iOS 而言) 或者 header「Authorization」(對非 iOS 而言) 的值。
    系統提供此服務前，先驗證此 session id 的值是否為有效的 session id， 若session id不存在或者已過期（預設有效期間為60分鐘，自上一次使用此session id調用服務的時間開始計算）， 則回傳錯誤訊息碼401（Unauthorized）。
  • 回傳的Response Header包含 field：「Change-Timestamp」， 其值為Desktop端資料異動的時間戳記（單位為millisecond），形態為長整數（long）。
