模組 檔案目錄
說明 向repository查詢一個或多個檔案上傳狀態。回傳結果可以是：
  • success: 成功
  • failure: 失敗
  • processing: 處理中
回傳值 更新上傳資料的結果。
URL POST http://<domain>:<port>/crepo/directory/dcupload2
參數範例（格式：application/json）
[
    {
        "transferKey" : "OTFDRjUxNzjglQUMlRTglQTklQTYuUE5HKzEzOTI2MjEwMjQzNDc=1",
        "status" : "confirming"
    },
    {
        "transferKey" : "OTFDRjUxNzjglQUMlRTglQTklQTYuUE5HKzEzOTI2MjEwMjQzNDc=2",
        "status" : "processing"
    },
    {
        "transferKey" : "OTFDRjUxNzjglQUMlRTglQTklQTYuUE5HKzEzOTI2MjEwMjQzNDc=3",
        "status" : "confirming"
    },
    {
        "transferKey" : "OTFDRjUxNzjglQUMlRTglQTklQTYuUE5HKzEzOTI2MjEwMjQzNDc=4",
        "status" : "processing"
    }
]
參數值說明
名稱 類型 說明
必要參數
transferKey string 上傳檔案key，同上傳檔案服務的參數「upkey」
非必要參數
status string 目前的上傳結果，做為repository回傳時參考用，可以不提供。此值可以是：
  • success: 成功
  • failure: 失敗
  • processing: 處理中
  • canceling: 取消中
  • not_found: 找不到這筆上傳資料
除非是為了特殊原因，需要再次確認檔案上傳結果，否則不建議查詢檔案上傳狀態為「success」或「failure」。
回傳值範例（格式：application/json）
[
    {
        "transferKey" : "OTFDRjUxNzjglQUMlRTglQTklQTYuUE5HKzEzOTI2MjEwMjQzNDc=1",
        "status" : "success"
    },
    {
        "transferKey" : "OTFDRjUxNzjglQUMlRTglQTklQTYuUE5HKzEzOTI2MjEwMjQzNDc=2",
        "status" : "not_found"
    },
    {
        "transferKey" : "OTFDRjUxNzjglQUMlRTglQTklQTYuUE5HKzEzOTI2MjEwMjQzNDc=3",
        "status" : "failure"
    },
    {
        "transferKey" : "OTFDRjUxNzjglQUMlRTglQTklQTYuUE5HKzEzOTI2MjEwMjQzNDc=4",
        "status" : "processing"
    }
]
回傳值說明

收到幾筆transfer key，就會回傳幾筆transfer key。沒找到的其狀態標示為「not_found」。

名稱 類型 說明
必要參數
transferKey string 上傳檔案key，同上參數「transferKey」
status string 檔案上傳結果，此值可以是：
  • success: 成功
  • failure: 失敗
  • processing: 處理中
  • device_uploaded_but_unconfirmed（檔案已經上傳至 server，但是還沒有完全送達 desktop。）
  • not_found: 找不到這筆上傳資料
若值為「processing」，表示desktop端尚未處理完成此筆檔案上傳。應稍後再調用此服務查詢此筆檔案上傳結果。 若為「not_found」，表示此檔案尚未上傳，或者上傳後，server尚未新增此筆上傳資料時。
非必要參數
（無）
回傳錯誤訊息（HTTP Status）
訊息碼 說明
400 指定的transferKey不存在或status值不是「success」或「failure」。
401 session 已過期（預設有效期間為 60 分鐘，自上一次使用此 session id 調用服務的時間開始計算）。
收到此回傳值，前端應調用服務 loginWithSessionId 取回新的 session id 後再重新調用此服務。
403 session 不存在。或者該 session 所對應的用戶不存在。
收到此回傳值，前端應提示用戶使用 Facebook Account Login 重新登入。
補充說明
必須先登入系統，取得有效的session id，以作為此服務的 header「fsi」(對 iOS 而言) 或者 header「Authorization」(對非 iOS 而言) 的值。
系統提供此服務前，先驗證此 session id 的值是否為有效的 session id， 若session id不存在或者已過期（預設有效期間為60分鐘，自上一次使用此session id調用服務的時間開始計算）， 則回傳錯誤訊息碼401（Unauthorized）。
