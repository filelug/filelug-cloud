模組 檔案目錄
說明 建立多檔上傳前的資料。包含子目錄、上傳摘要以及上傳通知類型。
回傳值 更新上傳資料的結果。

參數值說明
名稱 類型 說明
必要參數
upload-group-id string 多檔上傳檔案的群組編號。一個上傳群組編號，可以關聯一個或多個upload transfer key。
upload-keys string 上傳檔案key，同上傳檔案服務的參數「upkey」，但是可以一個以上的值。
upload-dir string 此上傳群組檔案儲存目錄。若子資料夾類型不是「0」，則此值包含子資料夾。
subdirectory-type integer 子資料夾類型。有效值如下：
  • 0 -- 沒有子資料夾（預設值）
  • 1 -- 目前時間
  • 2 -- 自訂的名稱
  • 3 -- 目前時間 ＋ 自訂的名稱
  • 4 -- 自訂的名稱 ＋ 目前時間
description-type integer 摘要說明類型。有效值如下：
  • 0 -- 無（預設值）
  • 1 -- 上傳檔案名稱清單
  • 2 -- 自訂上傳說明
  • 3 -- 自訂上傳說明 + 上傳檔案名稱清單
notification-type integer 上傳通知類型。有效值如下：
  • 0 -- 不通知
  • 1 -- 個別通知
  • 2 -- 整批通知（預設值）
非必要參數
subdirectory-value string 子資料夾名稱。若subdirectory-type為「0」，則此值不存在。
description-value string 摘要說明內容。若description-type值為「0」，則此值不存在。 若description-type值為「1」或「3」，此值會包含上傳檔案名稱列表，repository不負責建立檔案名稱列表資訊。

回傳錯誤訊息（HTTP Status）
訊息碼 說明
400 可能的原因如下，詳細需見錯誤訊息：
  • 指定的upload-key已存在。
  • subdirectory-type為「0」，且upload-dir資料夾不存在。
  • subdirectory-type不為「0」，且upload-dir資料夾的父資料夾不存在。
  • subdirectory-type不為「0」，且subdirectory-value的值不存在或者為空值。
  • subdirectory-type為「0」，且description-type不為「0」。
  • description-type不為「0」，且decription-value的值不存在或者為空值。。
401 session 已過期（預設有效期間為 60 分鐘，自上一次使用此 session id 調用服務的時間開始計算）。
收到此回傳值，前端應調用服務 loginWithSessionId 取回新的 session id 後再重新調用此服務。
403 session 不存在。或者該 session 所對應的用戶不存在。
收到此回傳值，前端應提示用戶使用 Facebook Account Login 重新登入。
466 Device端軟體版本較舊，Desktop版本較新。 Device收到此錯誤碼，應該提示使用者更新Device APP。
503 server 與 desktop 間尚未建立連線。
補充說明
必須先登入系統，取得有效的session id，以作為此服務的 header「fsi」(對 iOS 而言) 或者 header「Authorization」(對非 iOS 而言) 的值。
系統提供此服務前，先驗證此 session id 的值是否為有效的 session id， 若session id不存在或者已過期（預設有效期間為60分鐘，自上一次使用此session id調用服務的時間開始計算）， 則回傳錯誤訊息碼401（Unauthorized）。
