rename（尚未實作）
回上一層
模組 檔案目錄
說明 更改目錄或檔案名稱
回傳值 更名後的目錄或檔案路徑
URL POST http://<domain>:<port>/crepo/directory/rename
參數範例（格式：application/json）
{
    "path" : "/Users/albert2013/Downloads/temp/OriginalFilename.txt",
    "filename" : "UpdatedFilename.txt"
}
參數值說明
名稱 類型 說明
必要參數
path string 目錄或檔案更名前的路徑。此路徑值不須要encoded。
URL encoding的規則詳見：HTML URL Encoding Reference，其中的「URL Encoding Functions」可用來encode URL
filename string 目錄或檔案待更新的名稱。此路徑值不須要encoded。
URL encoding的規則詳見：HTML URL Encoding Reference，其中的「URL Encoding Functions」可用來encode URL。
非必要參數
（無）
回傳值範例（格式：application/json）
{
    "oldPath" : "/Users/albert2013/Downloads/temp/OriginalFilename.txt",
    "newPath" : "/Users/albert2013/Downloads/temp/UpdatedFilename.txt",
    "oldFilename" : "OriginalFilename.txt",
    "newFilename" : "UpdatedFilename.txt"
}
回傳值說明
屬性名稱 類型 說明
必要屬性
newPath string 目錄或檔案更名後的路徑。此路徑值不須要encoded。
URL encoding的規則詳見：HTML URL Encoding Reference，其中的「URL Encoding Functions」可用來encode URL
非必要屬性
（無）
回傳錯誤訊息（HTTP Status）
訊息碼 說明
400 下列原因之一造成：
  • 未指定目錄或檔案更名前的路徑（即 path 值）
  • 未指定目錄或檔案待更新的名稱（即 filename 值）
401 未提供使用者session id，或者該session id已經過期。
session id 必須作為 http request header「fsi」或者 「Authorization」值。詳見下方補充說明。
403 使用者對指定目錄或檔案沒有更名的權限
409 待更新的名稱已存在於作業系統相對目錄中
503 server 與 desktop 間尚未建立連線。
補充說明
  • 必須先登入系統，取得有效的session id，以作為此服務的 header「fsi」(對 iOS 而言) 或者 header「Authorization」(對非 iOS 而言) 的值。
    系統提供此服務前，先驗證此 session id 的值是否為有效的 session id， 若session id不存在或者已過期（預設有效期間為60分鐘，自上一次使用此session id調用服務的時間開始計算）， 則回傳錯誤訊息碼401（Unauthorized）。
  • 回傳的Response Header包含 field：「Change-Timestamp」， 其值為Desktop端資料異動的時間戳記（單位為millisecond），形態為長整數（long）。
