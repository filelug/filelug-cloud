findByPath
回上一層
模組 檔案目錄
說明 取得指定目錄或檔案的資訊
回傳值 指定目錄或檔案的資訊
URL POST http://<domain>:<port>/crepo/directory/find
參數範例（格式：application/json）
{
    "path" : "C:\\Documents and Settings\\Administrator\\My Documents\\My Pictures\\IDS258-001.png",
    "calculateSize" : true
}
參數值說明
名稱 類型 說明
必要參數
path string 目錄或檔案的絕對路徑。此路徑值不須要encoded。
URL encoding的規則詳見：HTML URL Encoding Reference，其中的「URL Encoding Functions」可用來encode URL
非必要參數
calculateSize boolean 值為true或false。 是否顯示目錄或檔案大小（size）。 目錄大小顯示該目錄下各階層子目錄與檔案大小總和。預設值為false，即不顯示。
回傳值範例（目錄、顯示大小）
{
    "symlink":false,
    "name":"Live Music Collection",
    "parent":"/Users/clopuccino/Music/iTunes/iTunes Media/Music",
    "readable":true,
    "writable":true,
    "executable":true,
    "displaySize":"10.59 MB",
    "hidden":false,
    "lastModified":"2013/03/08 14:51:55 GMT+0800",
    "type":"DIRECTORY",
    "contentType":"application/directory"
}
回傳值範例（檔案、顯示大小）
{
    "symlink":false,
    "name":"Cookbook.pdf",
    "parent":"/Users/clopuccino/Downloads",
    "readable":true,
    "writable":true,
    "executable":false,
    "displaySize":"7 MB",
    "hidden":false,
    "lastModified":"2013/03/31 08:32:55 GMT+0800",
    "type":"FILE",
    "contentType":"application/pdf"
}
回傳值說明
屬性名稱 類型 說明
必要屬性
symlink boolean 是否為symbolic link。對Windows作業系統而言，此值永遠是false。
name string 檔案或子目錄名稱。
parent string 父目錄。亦即參數path經過URL decode後的值
readable boolean 是否有讀的權限
writable boolean 是否有寫的權限
executable boolean 是否有執行的權限
hidden boolean 是否為隱藏檔
lastModified string 最後修改日期。格式為「yyyy/MM/dd HH:mm:ss 'GMT'Z」
type string 目錄或檔案。目錄值以「DIRECTORY」結尾、檔案值以「FILE」結尾。 有效值包含：
  • FILE,
  • DIRECTORY,
  • WINDOWS_SHORTCUT_FILE,
  • WINDOWS_SHORTCUT_DIRECTORY,
  • UNIX_SYMBOLIC_LINK_FILE,
  • UNIX_SYMBOLIC_LINK_DIRECTORY,
  • MAC_ALIAS_FILE,
  • MAC_ALIAS_DIRECTORY
contentType string type值以「DIRECTORY」時，此值永遠是「application/directory」。 若type值以「FILE」結尾時，此值依據檔案內容形態決定，詳見MIME on Wikipedia 。
非必要屬性
displaySize string 目錄或檔案大小（size）。 目錄大小顯示該目錄下各階層子目錄與檔案大小總和。
回傳錯誤訊息（HTTP Status）
訊息碼 說明
400 指定的目錄或檔案路徑不存在、或者calculateSize不為true/false。
401 session 已過期（預設有效期間為 60 分鐘，自上一次使用此 session id 調用服務的時間開始計算）。
收到此回傳值，前端應調用服務 loginWithSessionId 取回新的 session id 後再重新調用此服務。
403 session 不存在。或者該 session 所對應的用戶不存在。
收到此回傳值，前端應提示用戶使用 Facebook Account Login 重新登入。
503 server 與 desktop 間尚未建立連線。
補充說明
必須先登入系統，取得有效的session id，以作為此服務的 header「fsi」(對 iOS 而言) 或者 header「Authorization」(對非 iOS 而言) 的值。
系統提供此服務前，先驗證此 session id 的值是否為有效的 session id， 若session id不存在或者已過期（預設有效期間為60分鐘，自上一次使用此session id調用服務的時間開始計算）， 則回傳錯誤訊息碼401（Unauthorized）。
