模組 認證授權
說明 修改使用者密碼。
回傳值 使用者管理的所有電腦名稱。電腦名稱之間以分行符號「"\n"」分隔。最後以以分行符號「"\n"」結尾。 若使用者沒有管理任何電腦。則回傳空值。

回傳使用者管理的所有電腦名稱的目的在提醒前端使用者必須到這些電腦修改密碼後才能連線。

回傳值型態為：text/plain; charset=utf-8
URL POST http://<domain>:<port>/crepo/user/password
參數範例（格式：application/json）
{
    "old-passwd" : "81dc9bdb52d04dc20036dbd8313ed055",
    "new-passwd" : "81dc9bdb52d04dc20036dbd8313ed066"
}
參數值說明
名稱 類型 說明
old-passwd string 原有使用者密碼。此密碼為SHA-256加密後的密碼
new-passwd string 更新後使用者密碼。此密碼為SHA-256加密後的密碼
非必要參數
無
回傳值範例（格式：text/plain）
                連線電腦名稱1, 連線電腦名稱2, 連線電腦名稱3, 連線電腦名稱4
回傳值說明
屬性名稱 類型 說明
該用戶曾經連線過的電腦名稱，各個名稱間以「, 」（英文逗號加上一格空白）分隔。
回傳錯誤訊息（HTTP Status）
訊息碼 說明
400 可能的情況如下：
  • 未提供使用者舊密碼
  • 未提供使用者新密碼
  • 使用者帳號不存在
  • 舊密碼與既有的不相同
401 使用者session不存在或過期。
補充說明
  • 必須先登入系統，取得有效的session id，以作為此服務的 header「fsi」(對 iOS 而言) 或者 header「Authorization」(對非 iOS 而言) 的值。
    系統提供此服務前，先驗證此 session id 的值是否為有效的 session id， 若session id不存在或者已過期（預設有效期間為60分鐘，自上一次使用此session id調用服務的時間開始計算）， 則回傳錯誤訊息碼401（Unauthorized）。
