參數值說明
名稱 類型 說明
必要參數
account string 使用者編號
passwd string 使用者密碼。此密碼為SHA-256加密後的密碼
nickname string 使用者暱稱
verification string 驗證碼。用來確認此服務是由使用者經由授權的device執行。
locale string 使用者 locale
device-version string 此值不需要再提供，改紀錄於 device-token/filelug-version Filelug APP 的大版號，例如：1.0.0, 1.4.9，格式為： 
 ^\\d+(\\.\\d+)+
device-token/device-token string 該device使用的token
device-token/notification-type string Remote Notification Server的類型，有效值為：
  • APNS: For Apple
  • GCM: For Google
  • BAIDU: For Baidu
  • WNS: For Windows 8 (and later) or Windows Phone 8.1 app
  • NONE: Remote notification not supported
device-token/device-type string Device的作業系統類型，有效值為：
  • IOS: For apple iOS
  • OSX: For apple Mac OS X
  • WATCH_OS: For apple watchOS
  • WIN_PHONE: For Windows Phone/Pad
  • WIN_DESKTOP: For Windows desktop/server
  • ANDROID: For Android
  • CHROME: For ChromeOS or Chrome browser in desktop/server
  • LINUX_DESKTOP: For Linux desktop/server
  • MOBILE: For mobile devices such as phone and pad, if not applied to the values above
  • DESKTOP: For Desktop/Server, if not applied to the values above
device-token/device-version string device作業系統的版號，例如：8.3, 7.1，格式為： 
 ^\\d+(\\.\\d+)+
非必要參數
computer-id long 連線電腦編號。若不提供，則不檢查電腦端是否連線，也不通知電腦端連線。
showHidden boolean 是否顯示隱藏檔。若不設定此值，則以該使用者註冊時的值為準。 若 computer-id 不提供，此值也不需提供。
device.version string device 的 version number 資訊。 若 computer-id 不提供，此值也不需提供。
device.build string 此值不需要再提供，改紀錄於 device-token/filelug-build Filelug APP 的小版號
device-token/filelug-version string Filelug APP 的大版號，例如：1.0.0, 1.4.9，格式為： 
 ^\\d+(\\.\\d+)+
device-token/filelug-build string Filelug APP 的小版號，例如：2016.09.10.01，格式為：
yyyy.MM.dd.XX 最後兩位數為當天 commit 的第幾個 commit 的版本，例如 2016/01/22 當天第二個 commit 的版本為「2016.01.22.02」
device-token/badge-number integer badge number。此值必為正整數或者0。若此值不提供，且此device token不存在，則此值設為0。 若此值不提供，且此值存在，則此值不更新。

回傳值說明
屬性名稱 類型 說明
socket-connected boolean 說明 desktop 與 repository 的連線情況：
  • true：desktop 與 repository 連線已建立。
  • false：desktop 與 repository 連線未建立。 應調用服務：「Request Connection」(user/reconnect)要求desktop 建立連線。
lug-server-id string 指定連線的lug server子網域名稱。例如：「r1」表示指定連線的lug server為「r1.filelug.com」。 若 desktop 尚未連線，此值不存在。
回傳值為登入系統後的session id與使用者屬性， 若輸入參數包含 computer-id，則回傳值再包含與 desktop 相關的屬性。
後續的各項服務多需要此session id作為參數。
回傳錯誤訊息（HTTP Status）
訊息碼 說明
400 必要參數至少一個未提供。
401 使用者密碼與既有的不相同。
403 使用者帳號尚未註冊，或者尚未驗證。
465 Device端軟體版本較新，Desktop版本較舊。 Device收到此錯誤碼，應該提示使用者於電腦上下載並安裝新Desktop版本。 此錯誤只有當輸入參數包含 computer-id 時才可能發生。
466 Device端軟體版本較舊，Desktop版本較新。 Device收到此錯誤碼，應該提示使用者更新Device APP。 此錯誤只有當輸入參數包含 computer-id 時才可能發生。
467 使用者必須變更電話號碼才能繼續使用。
501 指定的電腦尚未安裝Filelug Desktop或尚未設定。 此錯誤只有當輸入參數包含 computer-id 時才可能發生。
補充說明
  • 取得的session id值，於後續調用其他服務時，會作為http header的「Authorization」值。
  • 提供 computer id 與不提供 computer id 的差別在於：
    不提供 computer id 的服務不考慮 desktop 的資訊，也不管其與 repository 之前連線的情況。
    提供 computer id 的服務回傳值包含「socket-connected」與「lug-server-id」值， 前者表示 desktop 是否已經與 repository 連線；後者為與 desktop 建立連線的 repository 主機的子網域名稱。
    不提供 computer id 的服務回傳值不包含「socket-connected」與「lug-server-id」的值。
