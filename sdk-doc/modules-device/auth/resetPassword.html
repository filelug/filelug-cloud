模組 認證授權
說明 重設使用者密碼。
回傳值 使用者管理的所有電腦名稱。電腦名稱之間以分行符號「"\n"」分隔。最後以以分行符號「"\n"」結尾。 若使用者沒有管理任何電腦。則回傳空值。

回傳使用者管理的所有電腦名稱的目的在提醒前端使用者必須到這些電腦修改密碼後才能連線。

回傳值型態為：text/plain; charset=utf-8
URL POST http://<domain>:<port>/crepo/user/reset-password
參數範例（格式：application/json）
{
    "account" : "9aaa3acb2bf8aa13533040cc4b24cec22089eba94b4d1c11f3a520656abdab5305b5b3f905b994ea10a24c16783fc340",
    "code" : "d04dc2003c5305b5b3f96abdab89eba05b994ea106dbd831",
    "new-passwd" : "81dc9bdb52d04dc20036dbd8313ed055",
    "verification" : "bf94b4d1c5305b5b3f96abdab89eba05b994ea10a249aaa32065c16",
    "locale" : "zh_TW"
}
參數值說明
名稱 類型 說明
account string 使用者編號
code string 加密後的使用者驗證碼。此驗證碼由使用者調用「sendResetPasswordSecurityCode」後，系統透過簡訊發送給使用者端。
new-passwd string 加密後的使用者新密碼。
verification string 系統驗證碼。用來確認此服務是由使用者經由授權的device執行，而不是機器在執行。
locale string 使用者 locale
非必要參數
無
回傳值範例（格式：text/plain）
                連線電腦名稱1, 連線電腦名稱2, 連線電腦名稱3, 連線電腦名稱4
回傳值說明
屬性名稱 類型 說明
該用戶曾經連線過的電腦名稱，各個名稱間以「, 」（英文逗號加上一格空白）分隔。
回傳錯誤訊息（HTTP Status）
訊息碼 說明
400 國家代碼、電話號碼、使用者驗證碼、新密碼與系統驗證碼至少一個未提供。
403 可能的情況：
  • 使用者不存在
  • 使用者已註冊但是尚未完成驗證
431 使用者驗證碼錯誤。
500 系統驗證碼錯誤。請求不是來自於授權的應用程式。
補充說明
  • 調用此服務前，須先調用「sendResetPasswordSecurityCode」以取得使用者驗證碼。
  • 重設密碼後，系統清除Repository上所有該使用者的client session。
