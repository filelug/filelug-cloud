模組 認證授權
說明 發送驗證碼給想要變更電子郵件地址的用戶。
回傳值 無
URL POST http://<domain>:<port>/crepo/user/change-email-code
參數範例（格式：application/json）
{
    "new-email" : "my-new-mail@example.com"
}
參數值說明
名稱 類型 說明
new-email string 更新後的電子郵件地址。
非必要參數
無
回傳錯誤訊息（HTTP Status）
訊息碼 說明
400 使用者驗證碼未提供。
403 session 不存在。或者該 session 所對應的用戶不存在。
收到此回傳值，前端提示用戶重新開啟應用程式，重新進行設定。
409 此電子郵件地址已經驗證過。系統比較用戶以驗證過的電子郵件與此電子郵件時，忽略大小寫的差異。 前端收到此訊息，要將該用戶的電子郵件記錄為此驗證過的電子郵件（使用小寫），標注此電子郵件已經驗證過。並告知用戶此電子郵件已經驗證過，並已自動變更用戶電子郵件為此。
補充說明
  • 必須先登入系統，取得有效的session id，以作為此服務的 header「fsi」(對 iOS 而言) 或者 header「Authorization」(對非 iOS 而言) 的值。
    系統提供此服務前，先驗證此 session id 的值是否為有效的 session id， 若session id不存在或者已過期（預設有效期間為60分鐘，自上一次使用此session id調用服務的時間開始計算）， 則回傳錯誤訊息碼401（Unauthorized）。
  • 系統發確認信至該電子郵件地址，調用服務「user/change-email」以變更電子郵件地址。
