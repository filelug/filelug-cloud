發送驗證碼給重設密碼用戶
回上一層
模組 認證授權
說明 發送驗證碼至用戶的電話號碼或者用戶的電子郵件地址，以供重設密碼用。 若用戶已被標示為「需要更新電話號碼」，發送驗證碼至用戶的電子郵件地址； 若不是，則發送簡訊驗證碼至用戶的電話號碼。
回傳值 驗證碼傳送目的地。 若發送到新電話號碼，回傳訊息為電話號碼（不含country id的值）； 若發送到使用者的電子郵件地址，回傳訊息為該電子郵件地址。
URL POST http://<domain>:<port>/crepo/user/reset-code
參數範例（格式：application/json）
{
    "account" : "9aaa3acb2bf8aa13533040cc4b24cec22089eba94b4d1c11f3a520656abdab5305b5b3f905b994ea10a24c16783fc340",
    "verification" : "bf94b4d1c5305b5b3f96abdab89eba05b994ea10a249aaa32065c16",
    "locale" : "zh_TW"
}
參數值說明
名稱 類型 說明
account string 使用者編號
verification string 驗證碼。用來確認此服務是由使用者經由授權的device執行，而不是機器在執行。
locale string 使用者 locale
非必要參數
無
回傳錯誤訊息（HTTP Status）
訊息碼 說明
400 使用者編號與驗證碼至少一個未提供。
403 使用者尚未註冊，或者使用者已註冊但是尚未完成驗證。
464 系統尚未支援此國家或地區發送簡訊。
468 使用者電子郵件地址未設定，無法寄送使用者驗證碼。
500 驗證碼錯誤。請求不是來自於授權的應用程式。
補充說明
  • 系統發確認簡訊至該電話號碼或者用戶的電子郵件地址， 使用者在重設密碼畫面輸入收到的簡訊驗證碼與新密碼後， 調用服務「resetPassword」以重設密碼。
  • 使用此服務不需要登入。
