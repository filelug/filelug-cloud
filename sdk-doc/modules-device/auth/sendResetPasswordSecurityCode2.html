發送內含驗證碼的電子郵件給重設密碼用戶
回上一層
模組 認證授權
說明 發送驗證碼至用戶的電子郵件地址，以供重設密碼用。
回傳值 驗證碼傳送的電子郵件地址。
URL POST http://<domain>:<port>/crepo/user/reset-code2
參數範例（格式：application/json）
{
    "country-id" : "TW",
    "phone" : "0975009123",
    "account" : "9aaa3acb2bf8aa13533040cc4b24cec22089eba94b4d1c11f3a520656abdab5305b5b3f905b994ea10a24c16783fc340",
    "email" : "my-new-mail@example.com"
    "verification" : "bf94b4d1c5305b5b3f96abdab89eba05b994ea10a249aaa32065c16",
    "locale" : "zh_TW"
}
參數值說明
非必要參數
名稱 類型 說明
verification string 驗證碼。用來確認此服務是由使用者經由授權的device執行，而不是機器在執行。
locale string 使用者 locale
非必要參數
account string 使用者編號。若此值不為空值，則忽略country-id與phone。 若此值為空值或者此值不存在，則使用country-id與phone。
country-id string 國家編號。 若 account 值不為空值，則此值忽略不處理。 若 account 為空值，則此值與 phone 皆不可為空值。
phone string 電話號碼。依據當地播打電話的規定，可能是「0」開頭。 若 account 值不為空值，則此值忽略不處理。 若 account 為空值，則此值與 country-id 皆不可為空值。
email string 用戶之前提供的電子郵件地址。 若不提供此資料，且採用國家編號與電話號碼（即使用者編號為空值）， 當系統使用電話號碼的方式找到的用戶數大於一個時，系統回傳409， 要求用戶提供email以供確認是哪一個用戶。
回傳值範例（格式：application/json）
{
    "account": "9aaa3acb2bf8aa13533040cc4b24cec22089eba94b4d1c11f3a520656abdab5305b5b3f905b994ea10a24c16783fc340",
    "email" : "jellyfish@example.com"
}
回傳值說明
屬性名稱 類型 說明
account string 使用者編號。
email string 寄送驗證碼至此電子郵件地址。
回傳錯誤訊息（HTTP Status）
訊息碼 說明
400 使用者編號、電話號碼（含國碼）至少一個未提供。 或者驗證碼未提供。
403 使用者尚未註冊，或者使用者已註冊但是尚未完成驗證。
409 此電話號碼關聯到一個以上的用戶，且輸入參數沒有用戶email以供確認。 前端收到此錯誤碼後，應要求用戶提供帳戶中的email address並重送此服務。
468 使用者電子郵件地址未設定，或者提供的電子郵件與該用戶當初設定的電子郵件不符， 無法寄送使用者驗證碼。
500 驗證碼錯誤。請求不是來自於授權的應用程式。
補充說明
  • 系統發確認簡訊至用戶的電子郵件地址， 使用者在重設密碼畫面輸入收到的驗證碼與新密碼後， 調用服務「resetPassword」以重設密碼。
  • 使用此服務不需要登入。
