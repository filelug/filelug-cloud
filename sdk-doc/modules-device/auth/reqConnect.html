Request Connection Between Repository and Desktop
回上一層
模組 認證授權
服務編號 1001
URL POST http://<domain>:<port>/crepo/user/reconnect
說明 要求建立Repository與Desktop間的連線
參數值說明
名稱 類型 說明
必要參數
（無）
非必要參數
（無）
回傳值範例（格式：application/json）
{
  "user-computer-id" : "9aaa3acb2bf8aa13533040cc4b24cec22089eba94b4d1c11f3a520656abdab5305b5b3f905b994ea10a24c16783fc340|79879797"
  "user-id" : "9aaa3acb2bf8aa13533040cc4b24cec22089eba94b4d1c11f3a520656abdab5305b5b3f905b994ea10a24c16783fc340",
  "computer-id" : 79879797,
  "computer-group" : "GENERAL",
  "computer-name" : "ALBERT'S WORKSTATION",
  "lug-server-id":"repo1"
}
回傳值說明
屬性名稱 類型 說明
必要屬性
user-computer-id string 使用者電腦編號。
user-id string 使用者帳號。
computer-id long 連線電腦編號
computer-admin-id string 連線電腦管理員的使用者編號
computer-name string 連線電腦名稱
computer-group string 連線電腦所屬群組
lug-server-id string 指定連線的lug server子網域名稱。例如：「r1」表示指定連線的lug server為「r1.filelug.com」。 基於測試用，若回傳的 lug-server-id 值為「aa」，表示直接連到「filelug.com」。
非必要屬性
（無）
回傳錯誤訊息（HTTP Status）
訊息碼 說明
400 找不到相關的使用者電腦資料。通常是因為desktop已經更改電腦名稱， client端收到此訊息碼應該要求使用者重新登錄。
401 session 已過期（預設有效期間為 60 分鐘，自上一次使用此 session id 調用服務的時間開始計算）。
收到此回傳值，前端應調用服務 loginWithSessionId 取回新的 session id 後再重新調用此服務。
403 session 不存在。或者該 session 所對應的用戶不存在。
收到此回傳值，前端應提示用戶使用 Facebook Account Login 重新登入。
補充說明
  1. 必須先登入系統，取得有效的session id，以作為此服務的 header「fsi」(對 iOS 而言) 或者 header「Authorization」(對非 iOS 而言) 的值。
    系統提供此服務前，先驗證此 session id 的值是否為有效的 session id， 若session id不存在或者已過期（預設有效期間為60分鐘，自上一次使用此session id調用服務的時間開始計算）， 則回傳錯誤訊息碼401（Unauthorized）。
  2. 當回傳狀態為 200 (OK) 時，表示要求成功，或者已經要求過，尚在處理中。
  3. 若發生其他錯誤，回傳狀態為 500（INTERNAL SERVER ERROR）。
