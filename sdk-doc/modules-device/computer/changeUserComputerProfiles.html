模組 電腦
說明 修改使用者電腦的設定值。包含：
  • 上傳檔案預設來源資料夾
  • 多檔上傳預設子資料夾類型
  • 多檔上傳預設子資料夾名稱
  • 多檔上傳預設備忘說明類型
  • 多檔上傳預設備忘說明內容
  • 多檔上傳預設通知類型
  • 下載檔案預設目的資料夾
  • 多檔下載預設子資料夾類型
  • 多檔下載預設子資料夾名稱
  • 多檔下載預設備忘說明類型
  • 多檔下載預設備忘說明內容
  • 多檔下載預設通知類型
回傳值 無
URL POST http://<domain>:<port>/crepo/computer/ucprofiles
參數範例（格式：application/json）
{
    "upload-directory" : "C:\Users\Administrator\Documents",
    "upload-subdirectory-type" : 4,
    "upload-subdirectory-value" : "西藏之旅",
    "upload-description-type" : 3,
    "upload-description-value" : "西藏布達拉宮",
    "upload-notification-type" : 2,
    "download-directory" : "/Storage/Emulated/0/Download",
    "download-subdirectory-type" : 4,
    "download-subdirectory-value" : "西藏之旅",
    "download-description-type" : 3,
    "download-description-value" : "西藏布達拉宮",
    "download-notification-type" : 2
}
參數值說明
名稱 類型 說明
必要參數
下列非必要參數必須至少提供一個。
非必要參數
upload-directory string 上傳檔案預設來源資料夾
upload-subdirectory-type integer 多檔上傳預設子資料夾類型。有效值如下：
  • 0 -- 無（不建立）
  • 1 -- 目前時間
  • 2 -- 自訂名稱
  • 3 -- 目前時間 ＋ 自訂名稱
  • 4 -- 自訂名稱 ＋ 目前時間
upload-subdirectory-value string 自訂子資料夾名稱。 此值僅包含「自訂子資料夾名稱」，不包含「目前時間」。
upload-description-type integer 上傳備忘說明類型。有效值如下：
  • 0 -- 無
  • 1 -- 上傳檔案名稱清單
  • 2 -- 自訂備忘說明
  • 3 -- 自訂備忘說明 + 上傳檔案名稱清單
upload-description-value string 自訂備忘說明內容。 此值僅包含「自訂備忘說明」，不包含「上傳檔案清單」。
upload-notification-type integer 上傳結果通知類型。有效值如下：
  • 0 -- 不通知
  • 1 -- 個別通知
  • 2 -- 整批通知
download-directory string 下載檔案預設目的資料夾
download-subdirectory-type integer 多檔下載預設子資料夾類型。有效值如下：
  • 0 -- 無（不建立）
  • 1 -- 目前時間
  • 2 -- 自訂名稱
  • 3 -- 目前時間 ＋ 自訂名稱
  • 4 -- 自訂名稱 ＋ 目前時間
download-subdirectory-value string 自訂子資料夾名稱。 此值僅包含「自訂子資料夾名稱」，不包含「目前時間」。
download-description-type integer 下載備忘說明類型。有效值如下：
  • 0 -- 無
  • 1 -- 下載檔案名稱清單
  • 2 -- 自訂備忘說明
  • 3 -- 自訂備忘說明 + 下載檔案名稱清單
download-description-value string 自訂備忘說明內容。 此值僅包含「自訂備忘說明」，不包含「下載檔案清單」。
download-notification-type integer 下載結果通知類型。有效值如下：
  • 0 -- 不通知
  • 1 -- 個別通知
  • 2 -- 整批通知
回傳錯誤訊息（HTTP Status）
訊息碼 說明
401 使用者session不存在、過期、 或者session id對應的user computer不存在。
session id 必須作為 http request header「fsi」或者 「Authorization」值。詳見下方補充說明。
補充說明
  • 必須先登入系統，取得有效的session id，以作為此服務的 header「fsi」(對 iOS 而言) 或者 header「Authorization」(對非 iOS 而言) 的值。
  • 由於必須取得使用者電腦相關資料，因此登入的session必須是包含computer資訊的。
  • 系統提供此服務前，先驗證此 session id 的值是否為有效的 session id， 若session id不存在或者已過期（預設有效期間為60分鐘，自上一次使用此session id調用服務的時間開始計算）， 則回傳錯誤訊息碼401（Unauthorized）。
