參數值說明
名稱 類型 說明
必要參數
sid integer 服務編號
account string 使用者帳號。
passwd string 使用者密碼。此密碼為SHA-256加密後的密碼
nickname string 使用者暱稱
verification string 驗證碼。用來確認此服務是由使用者經由授權的device執行，而不是機器在執行。
computer-id long 連線電腦編號
非必要參數
lug-server-id string 指定連線的lug server子網域名稱。例如：「r1」表示指定連線的lug server為「r1.filelug.com」。 若回傳值回空值或者lug-server-id不存在，則表示指定連線的lug server為「filelug.com」。
sysprops JSON object desktop（Ex. PC）主機上調用System.getProperties()的回傳值
回傳值範例（格式：application/json）
{
    "sid" : 1001,
    "status" : 200,
    "operatorId" : "TW-0975009123",
    "error" : "",
    "timestamp" : 1386505788544, // date time in millis
    "user" :
    {
        "account" : "9aaa3acb2bf8aa13533040cc4b24cec22089eba94b4d1c11f3a520656abdab5305b5b3f905b994ea10a24c16783fc340",
        "nickname" : "Jellyfish"
    }
}
回傳值說明
屬性名稱 類型 說明
必要參數
status integer 執行結果狀態碼。此值模仿一般 HTTP 回傳值，200 表示成功建立連線。非 2XX 表示建立連線失敗，可能情況如下所示。詳細錯誤訊息記錄於 error 參數值。
建立連線失敗狀態碼
訊息碼 說明
400 sid、使用者帳號、或密碼沒有提供。
401 使用者密碼錯誤。
403 使用者帳號尚未註冊或者已註冊但是尚未驗證。
404 sid值不是1001，該服務不支援。
460 此電腦不存在。
467 使用者必須變更電話號碼才能繼續使用。
sid integer 服務編號
非必要參數
error string 建立連線失敗的錯誤訊息。只有當 status 值不為 200 時才會有值。
operatorId string 與 repository 建立連線的 operator 帳號。通常為使用者帳號。
nickname string 使用者暱稱。此值不一定與Request參數的nickname一樣。此值是來自於repository資料庫中的值。 若別的Desktop使用此帳號，並更新nickname，則此值即回此更新後的nickname。
timestamp long 連線建立時間戳記。單位為毫秒（millisecond）。只有當 status 值為 200 時才會有值。
補充說明
  1. 與 repository 建立連線後取得的 session id 作為後續調用其他 repository 服務時，作為 JSON 的 sessionId 值。
  2. 一個帳號對應最多一個 session id，且最多只會有一條與 repository 的連線。
  3. 若連線過程發生錯誤，repository 會主動關閉與 desktop 之間的連線。
