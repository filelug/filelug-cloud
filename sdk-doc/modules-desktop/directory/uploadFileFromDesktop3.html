Upload File From Desktop (V3)
回上一層
模組 檔案目錄
URL POST http://<domain>:<port>/crepo/directory/supload3
說明 上傳檔案內容給 server，即 Device 端下載檔案時，Desktop 端使用此服務上傳檔案。 此服務與 V2 版本不同在於此服務需要在 header 中提供 desktop session id 與 file transfer key 兩項資訊。
回傳錯誤訊息（HTTP Status）
訊息碼 說明
400 沒有 file transfer key 或者找不到 file transfer key 對應的資料。
401 desktop session id 已經過期。
403 找不到 desktop session id 對應的 session 或 user 資訊。
500 尚未傳輸給前端時就發生其他錯誤。
503 device 端等待的 response 物件已經不可使用。可能是因為 device 端程序被取消或者某些因素造成該 response 物件已經被回傳而不能在被使用。desktop 端收到此回覆，不需要做事。
補充說明
  • 必須先登入系統，取得有效的session id，以作為此服務的 header「fsi」(對 iOS 而言) 或者 header「Authorization」(對非 iOS 而言) 的值。
  • 在 header「upkey」中提供 file transfer key 的值。
