Request Lug Server Connection
回上一層
模組 電腦
說明 取得目前適合連線的Lug Server子網域名稱。
回傳值 適合連線的Lug Server子網域名稱
URL POST http://<domain>:<port>/crepo/computer/dispatch2
無輸入參數
回傳值範例（格式：application/json）
{
  "lug-server-id":"repo1"
}
回傳值說明
屬性名稱 類型 說明
必要屬性
lug-server-id string 指定連線的lug server子網域名稱。例如：「r1」表示指定連線的lug server為「r1.filelug.com」。 基於測試用，若回傳的 lug-server-id 值為「aa」，表示直接連到「filelug.com」。
非必要屬性
（無）
回傳錯誤訊息（HTTP Status）
訊息碼 說明
403 session 不存在。或者該 session 所對應的用戶不存在。
收到此回傳值，前端提示用戶重新開啟應用程式，重新進行設定。
補充說明
  • 必須先登入系統，取得有效的session id，以作為此服務的 header「fsi」(對 iOS 而言) 或者 header「Authorization」(對非 iOS 而言) 的值。
  • 若 session id 對應的 session 已經過期，仍然視為正常而回傳 lug server id， 但是已經過期的 session 不更新其 last access timestamp。
