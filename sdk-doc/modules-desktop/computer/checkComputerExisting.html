Check Computer Existing
回上一層
模組 電腦
說明 確認指定電腦是否存在。當前端調用 server 服務，收到回傳值 403 時，再調用此服務確認該電腦確實已經不存在。
要注意的是，header 中的 session 必須是「不存在」。若該 session 已經存在，則回傳 400 錯誤碼。
回傳值 無內容。若 status code 為 200，表示該電腦仍然存在。若該電腦不存在，則回傳錯誤碼 460。
URL POST http://<domain>:<port>/crepo/computer/exist
參數範例（格式：application/json）
{
    "user-id" : "9aaa3acb2bf8aa13533040cc4b24cec22089eba94b4d1c11f3a520656abdab5305b5b3f905b994ea10a24c16783fc340",
    "computer-id" : 3837763637383939,
    "recoveryKey":"012336272652",
    "computer-group" : "GENERAL",
    "computer-name" : "ALBERT'S WORKSTATION",
    "verification" : "bf94b4d1c5305b5b3f96abdab89eba05b994ea10a249aaa3acb28aa13533040cc4b24cec2783fc3402011f3a52065c16",
    "locale" : "zh_TW",
}
參數值說明
名稱 類型 說明
必要參數
user-id string 使用者帳號。
computer-id long 電腦編號
recovryKey string 驗證電腦資訊必須要此作為憑證。為12位數的數字
computer-name string 連線電腦名稱
computer-group string 連線電腦所屬群組
verification string 驗證碼。用來確認此服務是由使用者經由授權的device執行，而不是機器在執行。
非必要參數
（無）
回傳錯誤訊息（HTTP Status）
訊息碼 說明
400 至少一個必要參數未提供、提供的 session id 已經存在、或者驗證碼錯誤。
460 指定電腦不存在。
補充說明
session id 置於 header「fsi」或者 header「Authorization」。
系統提供此服務前，先驗證此 session id 的值是否為存在， 若 session id 「已經存在」，則回傳錯誤訊息碼 400。
